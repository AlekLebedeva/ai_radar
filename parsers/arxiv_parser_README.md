# arXiv Parser

Парсер статей arXiv с поддержкой:

* выгрузки по категориям и датам;
* дедупликации записей;
* кэширования HTTP-запросов;
* отслеживания ранее выполненных запросов;
* автоматического восстановления после ошибок API;
* сохранения результатов в CSV;
* сохранения результатов в Excel;
* сохранения результатов в PostgreSQL.

---

# Архитектура

Общий поток выполнения:

```text
parse_arxiv_period()
        │
        ▼
 parse_arxiv()
        │
        ▼
 fetch_arxiv_papers()
        │
        ├── HttpCache
        ├── ProgressTracker
        ├── RequestTracker
        ├── request_with_backoff()
        └── parse_arxiv_entry()
        │
        ▼
 сохранение результатов
        ├── CSV
        ├── Excel
        └── PostgreSQL
```

---

# Основные компоненты

## parse_arxiv_period()

Точка входа для выгрузки за длительный период.

Функция автоматически разбивает период по месяцам и выполняет отдельную загрузку для каждого месяца.

Используется для больших объемов данных.

### Ответственность

* разбиение периода;
* контроль прогресса обработки месяцев;
* объединение результатов;
* формирование итоговой статистики.

---

## parse_arxiv()

Основная функция обработки одного временного интервала.

### Выполняет

* проверку доступности arXiv API;
* чтение robots.txt;
* инициализацию кэша;
* инициализацию трекеров;
* запуск загрузки статей;
* сохранение результатов.

---

## fetch_arxiv_papers()

Основной цикл получения статей.

Для каждой категории выполняется:

1. формирование запроса;
2. пагинация;
3. обработка ответа;
4. дедупликация;
5. преобразование данных в внутренний формат.

---

## parse_arxiv_entry()

Преобразует запись arXiv в унифицированный формат.

Возвращает словарь со всеми полями статьи.

# Система дедупликации

Используется класс:

```python
ProgressTracker
```

Для каждой статьи сохраняются:

* external_id
* SHA256(url + external_id)

При повторном запуске ранее обработанные статьи пропускаются.

Файл состояния:

```text
progress.json
```

(точный путь задаётся через Config)

---

# Система отслеживания запросов

Используется класс:

```python
RequestTracker
```

Назначение:

* хранение информации о ранее выполненных запросах;
* предотвращение повторного выполнения одинаковых запросов.

Каждый URL сохраняется как:

```text
SHA256(url)
```

Файл состояния:

```text
request_log.json
```

---

# HTTP-кэш

Используется класс:

```python
HttpCache
```

Кэшируется полный ответ API.

Ключ кэша:

```text
SHA256(url)
```

Формат хранения:

```json
{
  "url": "...",
  "status_code": 200,
  "content": "...",
  "cached_at": "..."
}
```

Просроченные записи удаляются автоматически.

---

# Обработка ошибок API

Для запросов используется:

```python
request_with_backoff()
```

Поддерживаются:

* 429 Too Many Requests
* 500 Internal Server Error
* 502 Bad Gateway
* 503 Service Unavailable
* 504 Gateway Timeout

Реализован:

* экспоненциальный backoff;
* поддержка Retry-After;
* случайный jitter.

---

# Robots.txt

Перед запуском выполняется чтение:

```text
https://arxiv.org/robots.txt
```

Если указан Crawl-delay, парсер автоматически использует его значение.

# Конфигурация

Все параметры находятся в объекте:

```python
Config
```

Основные настройки:

| Параметр          | Назначение                      |
| ----------------- | ------------------------------- |
| API_BASE_URL      | URL API arXiv                   |
| USER_AGENT        | User-Agent запросов             |
| MAX_RETRIES       | Количество повторных попыток    |
| BACKOFF_FACTOR    | Коэффициент увеличения задержки |
| API_TIMEOUT       | Таймаут запроса                 |
| CACHE_DIR         | Каталог кэша                    |
| CACHE_TTL_SECONDS | Время жизни кэша                |
| DATA_DIR          | Каталог результатов             |
| LOGS_DIR          | Каталог логов                   |
| CATEGORIES        | Категории по умолчанию          |

# Типовой запуск

Выгрузка за месяц:

```python
parse_arxiv(
    start_date="2026-05-01",
    end_date="2026-05-31"
)
```

Выгрузка за длительный период:

```python
parse_arxiv_period(
    start_date="2025-01-01",
    end_date="2025-12-31"
)
```

# Возможные проблемы

## Большое количество 429

Проверить:

* корректность User-Agent;
* настройки задержек;
* соблюдение robots.txt.

---

## Повторно не загружаются статьи

Проверить файл прогресса:

```text
progress.json
```

Для полной переиндексации:

* удалить progress-файл;
* отключить use_progress.

---

## Используются старые данные

Очистить кэш:

```python
http_cache.clear()
```

или использовать:

```python
clear_cache=True
```
